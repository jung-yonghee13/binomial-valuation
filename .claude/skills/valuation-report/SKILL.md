# 가치평가 및 평가보고서 산출 워크플로

계약정보와 평가 주요변수가 주어지면 가치평가 실행부터 평가보고서 산출까지 자동으로 수행한다.
평가자는 주요변수만 합리적으로 추정해 투입하고, 계산·검증·보고서 작성은 이 워크플로가 담당한다.

## 전체 흐름

```
[1] 계약정보 확인 → [2] 주요변수 수집 → [3] 가치평가 실행 → [4] 교차검증 → [5] 민감도 분석 → [6] 평가보고서 산출
```

## 1단계 — 계약정보 확인

`data/` 아래 계약 데이터 파일(JSON)을 로드하고 필수 필드를 검증한다.

필수 필드: 계약일, 투자자, 기초자산(발행회사·주식 종류·상장 여부), 대상주식수량,
콜옵션 행사가격, 거래종결일(만기), 행사방식(유럽형/미국형), 결제방식

- 필드가 누락되면 계산을 진행하지 말고 사용자에게 확인을 요청한다.
- 계약서 원문이 주어진 경우 위 필드를 추출·정리하여 JSON으로 저장한 뒤 사용자 확인을 받는다.

## 2단계 — 평가 주요변수 수집

사용자가 합리적으로 추정하여 제공하는 변수를 `data/valuation_inputs_template.json` 형식으로 정리한다.

| 변수 | 설명 | 출처 |
|------|------|------|
| 평가기준일 | 가치평가 기준 시점 | 사용자 지정 |
| 기초자산 가액 | 평가기준일 현재 주가(비상장 시 추정액) | 사용자 추정 |
| 행사가격 | 콜옵션 행사가 | 계약서 |
| 만기까지 기간 | 평가기준일 ~ 거래종결일 (연 단위 환산) | 계약서 + 평가기준일 |
| 무위험이자율 | 잔존만기 대응 국고채 수익률 등 (연속복리 환산) | 사용자 추정 |
| 배당 유무/배당수익률 | 기초자산 배당 정책 | 사용자 추정 |
| 주가 변동성 | 기초자산 연환산 변동성 (비상장 시 유사기업 변동성) | 사용자 추정 |

- 각 변수의 **추정 근거**를 반드시 함께 기록한다 (보고서 "주요변수 및 가정" 장에 그대로 수록됨).
- 변수가 누락된 채 평가를 요청받으면 기본값을 임의로 채우지 말고 사용자에게 요청한다.

## 3단계 — 가치평가 실행

`valuation/run_valuation.py`(구현 예정)로 CRR 이항모형 평가를 실행한다.

- 엔진: `valuation/binomial.py` — 벡터화된 역방향 귀납 (README 설계 원칙 준수)
- 위험중립확률과 할인계수는 동일한 연속복리 기준을 사용한다.
- 입력 검증 실패(무차익거래 조건 위반 등) 시 예외 메시지를 그대로 사용자에게 보고한다.
- 산출: 옵션 1주당 가치, 총 평가액(× 대상주식수량), 주가 트리·옵션가치 트리 요약

## 4단계 — 몬테카를로 교차검증

`valuation/monte_carlo.py`(구현 예정)로 동일 가정 하의 독립 계산을 수행한다.

- 동일한 위험중립 GBM 가정·파라미터로 경로 시뮬레이션 (시드 고정, 분산감소기법 적용)
- **합격 기준**: 이항모형 평가액이 몬테카를로 추정치의 95% 신뢰구간 내
- 경로 수·표준오차·신뢰구간을 기록하여 보고서에 수록
- 불합격 시 보고서를 산출하지 말고 원인(파라미터 불일치, 구현 오류 등)을 진단해 보고한다.

## 5단계 — 민감도 분석

주요변수 변동에 따른 평가액 변화를 계산한다 (보고서 수록용).

- 기초자산 가액 ±10%, 변동성 ±5%p, 무위험이자율 ±1%p (기본값, 사용자 조정 가능)
- 결과는 표 형태로 정리

## 6단계 — 평가보고서 산출

아래 목차 구조로 보고서를 생성하여 `reports/`에 저장한다.

```
제1장 Executive Summary
  1. 가치평가방법
  2. 평가대상 거래 정보          # 계약정보 요약 (1단계 산출물)
  3. 평가결과                    # 1주당 가치, 총 평가액, 교차검증 결과
  4. 주요변수 및 가정            # 2단계 변수 + 추정 근거
제2장 용역의 목적, 범위 및 한계
  1. 용역의 목적
  2. 용역의 범위 및 수행절차
  3. 용역의 한계
제3장 파생상품 가치평가
  1. 개요
  2. CRR모형 가치평가방법        # 모형 설명, 수식, 수치 설정(스텝 수 등)
  3. 거래 조건                   # 계약 조건 상세
  4. 평가결과                    # 트리 요약, 몬테카를로 교차검증, 민감도 분석
```

- 수치는 전부 파이썬 계산 결과에서 가져오고, 서술부만 생성한다. 계산되지 않은 숫자를 임의로 쓰지 않는다.
- 보고서에 몬테카를로 교차검증이 "계산의 정확성 검증"이며 "가정의 타당성 검증"과 구분됨을 명시한다.
- 가상 데이터 기반 보고서에는 검증용 가상 데이터임을 표기한다.

## 현재 구현 상태

| 구성요소 | 상태 |
|----------|------|
| 계약 데이터 (`data/sample_contract.json`) | 완료 (가상 데이터) |
| 주요변수 템플릿 (`data/valuation_inputs_template.json`) | 완료 |
| 이항모형 엔진 (`valuation/binomial.py`) | 구현 예정 |
| 몬테카를로 엔진 (`valuation/monte_carlo.py`) | 구현 예정 |
| 실행 진입점 (`valuation/run_valuation.py`) | 구현 예정 |
| 보고서 생성 | 구현 예정 |

구현 예정 스크립트가 없는 상태에서 이 스킬이 호출되면, 가능한 단계(1·2단계)까지 수행하고 나머지는 구현이 선행되어야 함을 알린다.

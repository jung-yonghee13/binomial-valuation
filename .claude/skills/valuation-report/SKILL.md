# 가치평가 및 평가보고서 산출 워크플로

계약정보와 평가 주요변수가 주어지면 가치평가 실행부터 평가보고서 산출까지 자동으로 수행한다.
평가자는 주요변수만 합리적으로 추정해 투입하고, 계산·검증·보고서 작성은 이 워크플로가 담당한다.

## 전체 흐름

```
[1] 계약정보 확인 → [2] 주요변수 수집 → [3] 가치평가 실행 → [4] 교차검증 → [5] 민감도 분석 → [6] 평가보고서 산출
```

## 1단계 — 계약정보 확인

`data/` 아래 계약 데이터 파일(JSON)을 로드하고 필수 필드를 검증한다.

필수 필드: 계약일, 투자자, 기초자산(발행회사·주식 종류·상장 여부), 대상주식수량,
콜옵션 행사가격, 거래종결일(만기), 행사방식(유럽형/미국형), 결제방식

- 필드가 누락되면 계산을 진행하지 말고 사용자에게 확인을 요청한다.
- 계약서 원문이 주어진 경우 위 필드를 추출·정리하여 JSON으로 저장한 뒤 사용자 확인을 받는다.

## 2단계 — 평가 주요변수 수집

변수를 `data/valuation_inputs_template.json` 형식으로 정리한다. 변동성과 무위험이자율은
아래 절차로 **자동 산출**하고, 나머지는 사용자가 합리적으로 추정하여 제공한다.

| 변수 | 설명 | 출처 |
|------|------|------|
| 평가기준일 | 가치평가 기준 시점 | 사용자 지정 |
| 기초자산 가액 | 평가기준일 현재 주가(비상장 시 추정액) | 사용자 추정 |
| 행사가격 | 콜옵션 행사가 | 계약서 |
| 만기까지 기간 | 평가기준일 ~ 거래종결일 (연 단위 환산) | 계약서 + 평가기준일 |
| 무위험이자율 | 국고채 spot rate (연속복리 환산) | **자동 산출 (2-B)** |
| 배당 유무/배당수익률 | 기초자산 배당 정책 | 사용자 추정 |
| 주가 변동성 | 피어그룹 역사적 변동성 평균 | **자동 산출 (2-A)** |

- 각 변수의 **추정 근거**를 반드시 함께 기록한다 (보고서 "주요변수 및 가정" 장에 그대로 수록됨).
- 자동 산출 변수도 산출 과정(피어그룹 구성, 데이터 기간, 계산 방법)을 근거로 기록한다.
- 변수가 누락된 채 평가를 요청받으면 기본값을 임의로 채우지 말고 사용자에게 요청한다.

### 2-A. 주가 변동성 자동 산출 (`valuation/volatility.py`, 구현 예정)

1. **피어그룹 확정**: 기초자산과 유사한 상장기업 5개사. 입력 파일의 `volatility_estimation.peer_group`을 사용한다.
   (실무에서는 전기 보통주 평가에 사용한 대용기업을 그대로 쓰는 경우가 많으나, 현 단계에서는 임의 선정한 피어그룹을 사용)
2. **주가 수집**: 각 피어의 **영업일 기준 일별 종가**를 API(pykrx, FinanceDataReader 등)로 수집.
   수집 기간은 평가기준일로부터 소급하여 `lookback_years`(기본 1년, 잔존만기 대응 기간으로 조정 가능).
3. **개별 변동성 계산**: 일별 로그수익률 → 표본표준편차 → `√252` 곱하여 연환산.
4. **평균**: 5개사 변동성의 산술평균을 기초자산 변동성으로 사용.
5. **기록**: 피어별 변동성·수집 기간·데이터 출처를 보고서 수록용으로 저장.

### 2-B. 무위험이자율 자동 산출 (`valuation/risk_free.py`)

1. **수익률 수집** — 우선순위:
   ① **Seibro 자동 수집(기본)**: 한국예탁결제원 증권정보포털의 '채권만기수익률'
      (출처: KIS자산평가) 데이터를 `valuation/seibro.py`가 직접 호출하여 국고채권
      3M/6M/9M/1Y/3Y/5Y/10Y/20Y 수익률을 수집한다. 무료 공개 조회라 API 키·브라우저·
      토큰이 모두 불필요하며, 휴일이면 직전 영업일로 자동 소급한다.
      `run_valuation`에서 `risk_free_rate: "auto"` + 곡선 파일 미지정 시 자동 실행.
   ② 사용자가 지정한 곡선 JSON 파일 (`risk_free_estimation.ytm_curve_file`)
   ③ **KOFIA 브라우저 수집(폴백, 2-B-a)**: KOFIA 채권정보센터는 공식 API가 없고
      WAF가 스크립트 접근(Playwright/requests)을 차단하므로, Seibro 장애 시에만
      사용자의 실제 Chrome을 쓰는 에이전트 브라우저 수집으로 대응한다.
2. **Spot rate 산출**: 만기별 YTM 곡선에서 부트스트래핑으로 spot rate 곡선을 산출.
3. **만기 대응**: 평가대상의 잔존만기에 대응하는 spot rate를 보간(선형 또는 스플라인)하여 선택.
4. **복리 환산**: 산출된 spot rate를 **연속복리 기준으로 환산**하여 트리 엔진에 투입
   (엔진의 복리 일관성 원칙과 정합).
5. **기록**: 수집 일자·만기별 수익률·산출된 spot rate를 보고서 수록용으로 저장.
6. **기간구조 반영(선택)**: 단일 spot rate 대신 `step_forward_rates()`로 스텝별
   선도이자율을 도출하고 `binomial.price_with_curve()`로 평가하면 수익률곡선의
   기간구조가 트리에 반영된다 (실무 평가모델 방식, `docs/valuation-logic-notes.md` 참조).

#### 2-B-a. KOFIA 국고채 수익률 브라우저 수집 절차 (폴백 — Seibro 장애 시에만)

KOFIA 채권정보센터는 공식 API가 없고 WAF가 스크립트 접근을 차단하므로,
에이전트가 사용자의 Chrome 브라우저로 페이지를 직접 열어 수익률을 읽어온다. 절차:

1. **도구 준비**: ToolSearch로 `mcp__claude-in-chrome__*` 도구 로드 →
   `tabs_context_mcp(createIfEmpty)` → 새 탭 사용.
2. **페이지 접속**: 최종호가수익률(시가평가 기준수익률) 페이지로 이동:
   `https://www.kofiabond.or.kr/websquare/websquare.html?w2xPath=/xml/mm/mm1002/BISMM1002.xml&divisionId=MBIS01000000000000&serviceId=`
3. **로딩 대기**: WebSquare 기반이라 로딩이 느리다. 5~10초 `wait` 후 재시도.
   **최초 사용 시 Chrome 확장의 사이트 권한 승인이 필요** — 도구가 계속 무응답이면
   사용자에게 확장 프로그램 사이드패널에서 kofiabond.or.kr 권한 승인을 요청한다.
4. **기준일 설정**: 조회일자를 평가기준일로 변경 후 조회 실행
   (기본값은 최근 영업일. 평가기준일이 휴일이면 직전 영업일 사용을 사용자에게 확인).
5. **값 추출**: `get_page_text` 또는 `read_page`로 표에서 **국고채** 행의
   만기별 수익률(%)을 읽는다. 기준은 **민평평균**(평가사 평균).
   만기 축: 3월/6월/9월/1년/1년6월/2년/2년6월/3년/4년/5년/7년/10년/15년/20년/30년/50년
   → 연 단위 [0.25, 0.5, 0.75, 1, 1.5, 2, 2.5, 3, 4, 5, 7, 10, 15, 20, 30, 50]
6. **검증**: 수익률을 소수로 변환(÷100)한 값이 0 ~ 0.2 범위인지, 만기가 오름차순인지 확인.
   벗어나면 잘못 읽은 것이므로 재추출한다.
7. **JSON 저장**: `data/ytm_curve_<평가기준일>.json`으로 저장:
   ```json
   {
     "date": "<평가기준일>",
     "source": "KOFIA 채권정보센터 최종호가수익률 (국고채, 민평평균) — 브라우저 수집",
     "collected_at": "<수집 시각>",
     "maturities": [0.25, 0.5, ...],
     "yields": [0.0286, 0.0279, ...]
   }
   ```
8. **연결**: 입력 파일의 `risk_free_estimation.ytm_curve_file`에 위 경로를 지정.
   이후 spot rate 부트스트래핑·선도이자율 산출은 파이썬(`risk_free.py`)이 수행한다.
9. **폴백**: 수집 실패(권한 미승인, 페이지 구조 변경 등) 시 사용자에게 해당 페이지의
   국고채 수익률을 수동으로 알려달라고 요청하고, 동일 형식의 JSON으로 저장한다.
   보고서에는 수집 방법(브라우저 수집/수동 입력)과 수집 일시를 기재한다.

## 3단계 — 가치평가 실행

`valuation/run_valuation.py`(구현 예정)로 CRR 이항모형 평가를 실행한다.

- 엔진: `valuation/binomial.py` — 벡터화된 역방향 귀납 (README 설계 원칙 준수)
- 위험중립확률과 할인계수는 동일한 연속복리 기준을 사용한다.
- **기간구조 할인(기본 권장)**: 입력의 `risk_free_estimation.discounting`을
  `"term_structure"`로 두면 수익률곡선 → 스텝별 선도이자율 → 스텝별 확률·할인이
  자동 적용된다 (실무 평가모델 방식). `numerics.step_unit: "weekly"`면 주 단위 트리.
  `"spot"`이면 만기 대응 spot rate 단일 할인.
- 입력 검증 실패(무차익거래 조건 위반 등) 시 예외 메시지를 그대로 사용자에게 보고한다.
- 산출: 옵션 1주당 가치, 총 평가액(× 대상주식수량), 주가 트리·옵션가치 트리 요약

## 4단계 — 몬테카를로 교차검증

`valuation/monte_carlo.py`(구현 예정)로 동일 가정 하의 독립 계산을 수행한다.

- 동일한 위험중립 GBM 가정·파라미터로 경로 시뮬레이션 (시드 고정, 분산감소기법 적용)
- **합격 기준**: 이항모형 평가액이 몬테카를로 추정치의 95% 신뢰구간 내
- 경로 수·표준오차·신뢰구간을 기록하여 보고서에 수록
- 불합격 시 보고서를 산출하지 말고 원인(파라미터 불일치, 구현 오류 등)을 진단해 보고한다.

## 5단계 — 민감도 분석

주요변수 변동에 따른 평가액 변화를 계산한다 (보고서 수록용).

- 기초자산 가액 ±10%, 변동성 ±5%p, 무위험이자율 ±1%p (기본값, 사용자 조정 가능)
- 결과는 표 형태로 정리

## 6단계 — 평가보고서 산출

아래 목차 구조로 보고서를 생성하여 `reports/`에 **PDF로 저장**한다.
(중간 산출물로 HTML을 생성한 뒤 PDF로 변환하는 파이프라인 사용. 파일명은 URL 인코딩 문제가 없도록 영문 사용: `valuation-report_<평가기준일>.pdf`)

```
제1장 Executive Summary
  1. 가치평가방법
  2. 평가대상 거래 정보          # 계약정보 요약 (1단계 산출물)
  3. 평가결과                    # 1주당 가치, 총 평가액, 교차검증 결과
  4. 주요변수 및 가정            # 2단계 변수 + 추정 근거
제2장 용역의 목적, 범위 및 한계
  1. 용역의 목적
  2. 용역의 범위 및 수행절차
  3. 용역의 한계
제3장 파생상품 가치평가
  1. 개요
  2. CRR모형 가치평가방법        # 모형 설명, 수식, 수치 설정(스텝 수 등)
  3. 거래 조건                   # 계약 조건 상세
  4. 평가결과                    # 트리 요약, 몬테카를로 교차검증, 민감도 분석
```

- 수치는 전부 파이썬 계산 결과에서 가져오고, 서술부만 생성한다. 계산되지 않은 숫자를 임의로 쓰지 않는다.
- 보고서에 몬테카를로 교차검증이 "계산의 정확성 검증"이며 "가정의 타당성 검증"과 구분됨을 명시한다.
- 가상 데이터 기반 보고서에는 검증용 가상 데이터임을 표기한다.

## 행동 강령 준수

모든 단계는 `docs/conduct-guidelines.md`(외부평가업무 행동 강령)를 따른다. 특히:

- 자동 수집 데이터(주가, 금리)는 출처·수집 일자를 결과에 보존한다.
- 보고서에는 행동 강령 4절의 필수 기재사항(평가기준일·목적, 목적 외 사용 제한,
  정보 신뢰의 한계, 미래 예측의 불확실성, 주요 가정과 추정 근거, 공개 제한)을 반드시 포함한다.
- 가상 데이터 기반 산출물에는 "검증용 가상 데이터" 표기를 유지한다.

## 현재 구현 상태

| 구성요소 | 상태 |
|----------|------|
| 계약 데이터 (`data/sample_contract.json`) | 완료 (가상 데이터) |
| 주요변수 템플릿 (`data/valuation_inputs_template.json`) | 완료 |
| 변동성 자동 산출 (`valuation/volatility.py`) | 완료 (FinanceDataReader) |
| 무위험이자율 산출 (`valuation/risk_free.py`) | 완료 — 단, KOFIA 자동 수집은 미구현. 수익률 곡선 JSON을 `risk_free_estimation.ytm_curve_file`로 지정 |
| 이항모형 엔진 (`valuation/binomial.py`) | 완료 (테스트 포함) |
| 몬테카를로 엔진 (`valuation/monte_carlo.py`) | 완료 (유럽형; LSMC 확장 예정) |
| 실행 진입점 (`valuation/run_valuation.py`) | 완료 |
| 보고서 생성 (`valuation/report.py`) | 완료 — 결과 JSON → HTML → PDF (Edge/Chrome headless 인쇄) |

실행 명령:
1. 평가: `python -m valuation.run_valuation --contract <계약.json> --inputs <입력.json> --output-dir results`
2. 보고서: `python -m valuation.report --contract <계약.json> --result results/valuation_result_<평가기준일>.json --output-dir reports`

결과 JSON에 평가액·교차검증·민감도·변수 산출 내역이 모두 포함되며, 보고서는 이 JSON의 수치만 사용한다.
보고서에는 행동 강령의 필수 기재사항(표지 면책, 용역의 한계, 가상 데이터 표기)이 자동 포함된다.

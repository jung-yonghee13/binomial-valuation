# 실무 평가모델 학습 노트 (콜옵션 파생상품 공정가치평가)

실제 콜옵션 공정가치평가 엑셀 실무 모델(비식별 처리됨)을 분석하여 학습한 계산 로직 정리.
본 프로젝트 엔진에 반영할 사항과 개선점을 함께 기록한다. 거래 특정 수치·당사자 정보는 제외.

## 1. 전체 구조 (엑셀 시트 구성)

| 시트 | 역할 |
|------|------|
| Call가치평가 | 기초정보(계약일·평가기준일·만기), 적용 변수, 옵션 조건, Call Tree 역산, 평가결과 |
| 주가트리 | u/d/p 파라미터, 주 단위 주가 트리 (전환·행사가격 조정 반영) |
| RF Spot rate | 국고채 만기수익률 → 분기 par bond 부트스트래핑 → spot rate 곡선 |
| Forward Rate_weekly | spot rate → 주 단위 선도이자율 (트리 할인용) |
| 주가변동성 | 피어(대용기업) 일별 종가 → 로그수익률 → 연환산 변동성 → 평균 |

## 2. 핵심 계산 관행 (실무 모델에서 확인)

### 트리 구성 — 주(週) 단위
- 스텝 길이 dt = 7일. 평가기준일부터 만기까지 주 단위 날짜 그리드 생성
- 만기연환산은 `YEARFRAC`, 경과일수/만기일수는 실제 일수 기준

### 변동성 (피어그룹)
- 피어 종가 약 180영업일 수집 → 일별 로그수익률 `LN(P_t/P_{t-1})`
- 연환산: 표준편차 × √250 (연 영업일 250일 가정)
- 피어별 변동성의 산술평균 적용
- 주간 변동성 환산: σ_week = σ_year / √365 × √7 (달력일 기준 스케일링)
- 주석 명시: "가치평가의 일관성을 유지하기 위하여 전기 보통주 평가에 사용된 대용기업을 준용함"

### 무위험이자율 (KOFIA 국고채 → spot → 선도)
1. KOFIA 채권정보센터(kofiabond.or.kr) 국고채 만기수익률(민평평균) 수집
   — 3M/6M/9M/1Y/1.5Y/2Y/2.5Y/3Y/4Y/5Y/7Y/10Y/15Y/20Y/30Y/50Y
2. 분기 수익률로 변환: `(1+y)^0.25 − 1`, 고시 만기 사이는 로그선형 보간
3. 분기 이표 par bond 현금흐름을 구성해 **부트스트래핑으로 spot rate 곡선** 산출
4. spot rate를 주 단위로 환산하고 **주별 선도이자율(forward rate)** 도출:
   `1 + f_i = DF(t_{i-1}) / DF(t_i)`
5. 트리의 각 스텝에 해당 주의 선도이자율을 적용

### 트리 파라미터
- 상승배수 u = exp(σ_week), 하락배수 d = 1/u (CRR)
- 위험중립확률을 **스텝별로** 계산: `p_i = (1 + f_i − d) / (u − d)`

### Call Tree 역방향 귀납 (엑셀 수식 구조)
```
V(j, t) = IF(j > t, 0,
    MAX( MAX(S(j,t) − K(t), 0),                        ← 즉시 행사가치
         EXP(−f_t) × [p_t·V(j+1,t+1) + (1−p_t)·V(j,t+1)] ))  ← 계속보유가치
```
- 미국형: 각 노드에서 행사 vs 계속보유 중 큰 값
- 행사가격 K(t)는 스텝별 행(별도 블록)에서 참조 — 조정(리픽싱 등) 반영 가능 구조
- 상장 시나리오별 기초가액 블록(전일종가/1주 평균/1개월 평균 등)과 Node별 확률 블록 존재

### 평가결과 산출 구조
```
Call옵션 수량 = 대상 주식수 × 행사범위(%) × 행사자별 행사비율(%)
평가액 = Call옵션 수량 × 개당 콜옵션가치 (트리 (0,0) 노드 값)
```

## 3. 본 프로젝트 반영 사항

| 실무 로직 | 반영 |
|-----------|------|
| spot rate 부트스트래핑 | `risk_free.bootstrap_spot_curve()` (기존) |
| **주별 선도이자율 도출** | `risk_free.step_forward_rates()` — DF 비율 방식 |
| **스텝별 이자율 트리 평가** | `binomial.price_with_curve()` — 스텝별 p_i·할인 적용 |
| 피어그룹 변동성 | `volatility.peer_group_volatility()` (기존) |
| 미국형 조기행사 | `price()` / `price_with_curve()`의 `american=True` |

## 4. 발견한 개선점 (엔진에서 수정한 사항)

1. **복리 혼용**: 실무 모델은 위험중립확률에 `(1 + f)`를 쓰면서 할인은 `EXP(−f)`를
   사용 — 주 단위에서는 오차가 미미하지만 이론적으로 비일관. 엔진은 성장배수
   `g_i = 1 + f_i`를 확률과 할인(1/g_i) 양쪽에 동일하게 적용한다.
2. **변동성 스케일링 혼용**: 연환산은 √250(영업일), 주간 환산은 √(7/365)(달력일)로
   기준이 섞여 있음. 엔진은 `σ_step = σ_year × √(step_years)`로 일관 적용하되,
   보고서 '주요변수 및 가정'에 스케일링 기준을 명시한다.
3. **무차익거래 검증 부재**: 엑셀은 p가 (0,1)을 벗어나도 계산이 진행됨.
   엔진은 스텝별로 `d < 1+f_i < u`를 검증하고 위반 시 예외를 발생시킨다.

## 5. 남은 자동화 과제 (로드맵 연계)

- KOFIA 국고채 수익률 자동 수집 (현재: 수동 곡선 JSON 입력)
- 스텝별 행사가격 스케줄(조정·리픽싱) 지원 — payoff를 시점 의존 함수로 확장
- 상장 시나리오별 기초가액 처리 (전일종가/평균종가 등 계약 조건별)
- run_valuation에 기간구조 모드 통합 (현재는 엔진 함수 제공 단계)
- 계약서 문서 분석 → 계약정보 JSON 자동 추출 → 평가 자동 실행 에이전트
